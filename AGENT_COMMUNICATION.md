## 🎯 核心原则

### 1. 使用 `sessions_send` 进行内部通信
- ✅ **正确**: 用 `sessions_send` 发送消息给其他 Agent
- ❌ **错误**: 在飞书群@机器人、发飞书私信给其他 Agent

### 2. 禁止外部通信暴露
- ❌ 禁止用 `feishu_im_user_message` 给其他 Agent 发消息
- ❌ 禁止在飞书群聊中暴露 Agent 间协调过程
- ✅ Agent 间协调全部通过 `sessions_send` 内部完成

### 3. 消息格式标准化
```markdown
[消息类型] 来源：Agent 名 | 优先级：🔴/🟡/🟢 | 内容：xxx
```

**示例**:

```

---

### 2. 学习汇报（Agent → 年年/全队）

**格式**:
```
[学习汇报] 来源：Agent 名 | 领域：xxx | 价值：🔴/🟡/🟢 | 核心收获：xxx | 用在哪：xxx | 预期改善：xxx

**示例**:
```

**接收方**:
- 🔴 高价值 → 发送给年年 + 写入 SHARED_KNOWLEDGE.md
- 🟡 中价值 → 发送给年年
- 🟢 基础价值 → 可自行记录

---

### 3. 求助请求（Agent → Agent）

**格式**:
```
[求助] 来源：Agent 名 | 目标：Agent 名 | 问题：xxx | 紧急度：高/中/低
```

**示例**:
```

**响应要求**:
- 高紧急度 → 1 小时内响应
- 中紧急度 → 4 小时内响应
- 低紧急度 → 24 小时内响应

## 🛠️ 技术实现

### sessions_send 用法（正确方式）

**重要**: 必须使用 `sessionKey` 参数，不要用 `agentId` 或 `label`！

```python
# ✅ 正确：使用 sessionKey
sessions_send(
    sessionKey="agent:taiyi:main",  # 目标 Agent 的 main session
    message="[任务分发] 来源：年年 | 优先级：🔴 | 任务：xxx"
)

# ✅ 正确：使用 sessionKey 发送给 cron session
sessions_send(
    sessionKey="agent:taiyi:cron:xxx",  # 目标 cron session
    message="[学习汇报] 来源：太一 | ..."
)

# ❌ 错误：不要用 agentId
sessions_send(agentId="taiyi", ...)  # 会报错！

# ❌ 错误：不要用 label
sessions_send(label="agent:taiyi:main", ...)  # 会报错！
```

## ⚠️ 禁止行为

### ❌ 绝对禁止

1. **用飞书私信联系其他 Agent**
   - 错误：`feishu_im_user_message` 发给玄机
   - 正确：`sessions_send` 发给玄机

2. **在飞书群暴露 Agent 协调**
   - 错误：在群里@望舒说"PRD 写好了吗"
   - 正确：`sessions_send` 私下问望舒

3. **直接调用主人账号发消息**
   - 错误：用 `feishu_im_user_message` 以主人身份发消息
   - 正确：向年年汇报，由年年决定如何通知主人

4. **跨 session 直接调用工具**
   - 错误：直接操作其他 Agent 的文件
   - 正确：发送请求，让对方自己操作

## 🔄 通讯流程

### 任务分发流程
```
年年收到主人需求
    ↓
拆解任务 → sessions_send 分发给对应 Agent
    ↓
Agent 接收 → 确认 → 开始执行
    ↓
定期进度汇报 → sessions_send 回传年年
    ↓
任务完成 → 验收 → 归档
```

### 学习汇报流程
```
Agent 完成学习
    ↓
写入 lessons.md
    ↓
sessions_send 发送学习汇报给年年
    ↓
年年记录到 SHARED_KNOWLEDGE.md
    ↓
年年反哺给相关 Agent
    ↓
其他 Agent 吸收应用
```

### 求助流程
```
Agent A 遇到问题
    ↓
sessions_send 发送 [求助] 给 Agent B
    ↓
Agent B 响应 → 提供帮助
    ↓
问题解决 → Agent A 发送 [感谢]
    ↓
年年记录到 SHARED_KNOWLEDGE.md
```

## 🎯 最佳实践

### ✅ 推荐做法

1. **消息简洁** - 用模板，不啰嗦
2. **优先级明确** - 标注🔴/🟡/🟢
3. **截止时间** - 所有任务都有 deadline
4. **验收标准** - 任务分发时明确什么是完成
5. **及时反馈** - 收到消息先确认，完成再汇报

### ❌ 避免做法

1. **长篇大论** - 消息超过 500 字
2. **无优先级** - 不标注紧急程度
3. **无截止** - 不说什么时候完成
4. **已读不回** - 收到消息不确认
5. **事后汇报** - 完成了才说，中间不更新

## 🔒 安全规则

1. **不泄露主人隐私** - Agent 间通信也不得泄露
2. **不越权操作** - 只能操作自己权限内的文件
3. **不伪造身份** - 消息必须真实来源
4. **不绕过审核** - 敏感操作需年年确认
