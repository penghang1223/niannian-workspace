## 🎟️ 内测邀请系统技术实现文档

**时间：** 2026-03-10 15:31-16:24  
**主题：** 内测邀请系统技术实现详解

### 核心内容
- **技术栈**：Node.js v22+ + 飞书 Open API + JSON 文件存储
- **零第三方依赖**：所有功能使用 Node.js 原生模块（https、fs、path）
- **核心文件**：
  - invite_beta_user.js - 添加用户、设置调用限制
  - send_beta_invite.js - 通过飞书 API 发送邀请消息
  - beta_users.json - 内测用户列表存储
  - rate_limit_manager.js - 滑动时间窗口频率限制
  - openclaw_multi_role.js - 白名单管理、权限验证

### 技术实现细节
1. **飞书 API 集成**
   - 获取 token：POST /auth/v3/tenant_access_token/internal
   - 发送消息：POST /im/v1/messages
   - 使用原生 https 模块，Promise 封装异步操作

2. **白名单机制**
   - 使用 Set 存储用户 ID
   - isBetaUser() 验证权限
   - 数据持久化到 beta_users.json

3. **频率限制**
   - 三级限制：每日（200 次）、每小时（30 次）、每分钟（8 次）
   - 滑动时间窗口算法
   - 记录每次请求时间戳，动态计算

4. **关键词激活**
   - 监听用户回复"接受内测邀请"
   - 自动激活完整功能权限
   - 初始化用户配置

### 安全机制
- 白名单验证
- 频率限制（防止 API 滥用）
- 容量控制（默认 10 人）
- 权限隔离（内测/普通用户分离）

## 🏗️ OpenClaw 架构详解

**时间：** 2026-03-10 14:08-14:31  
**主题：** OpenClaw 完整架构讲解

### 核心层级
1. **渠道层** - 飞书/钉钉/微信等通讯平台
2. **网关层** - Gateway（端口 18789，消息路由、认证、会话管理）
3. **Agent 层** - 年年/岁岁/开发/产品/测试等 AI 智能体
4. **工具层** - web_search/message/tts等扩展工具
5. **记忆层** - MEMORY.md + memory/ + 向量搜索
6. **模型层** - GLM/Qwen/Claude等大模型

### 消息流转示例
```
主人发消息 → 飞书接收 → Gateway 路由 → 年年处理 → 
调用工具/子代理 → 汇总结果 → 发送回复
```

### 配置文件
- openclaw.json - 主配置（agents、channels、bindings）
- SOUL.md - 角色定义
- USER.md - 主人信息
- MEMORY.md - 长期记忆
- HEARTBEAT.md - 心跳任务
