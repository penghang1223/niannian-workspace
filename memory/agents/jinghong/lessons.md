## 📚 今日学习

### 2026-04-02

#### 学习闭环 #1

##### 学了什么
- **领域**：小说场景结构 & 张力升级技法
- **来源**：
  - https://www.novlr.org/the-reading-room/the-fichtean-curve-how-to-escalate-tension-and-conflict/
  - https://writersdigestonline.com/scene-building/
- **核心知识点**：
  1. **Fichtean Curve（费希特曲线）**：不设传统三幕式的漫长铺垫，而是从激励事件开始直接进入一系列递增的危机（crises），每个危机都是一个小高潮，持续推高张力直到巅峰，然后下降解决。特别适合悬疑、宫斗、言情。
  2. **Goal-Conflict-Disaster 场景三件套**：每个场景必须有明确目标 → 遇到阻碍 → 产生结果。四种结果类型：
     - **Yes**：成功（少用，多了故事太顺）
     - **Yes, but**：成功但有代价/新问题（最常用）
     - **No**：失败，需要新方案
     - **No, and furthermore**：失败且更糟（高张力点）
  3. **Sequel（后续反应）**：场景之间的缓冲带，结构为 Reaction → Dilemma → Decision。先让角色感受情绪，再权衡选项，再做出决定（决定成为下一个场景的目标）。
  4. **Try-Fail Cycle**：一个场景内多次尝试→失败→再尝试，制造努力感和真实感。

##### 有用吗？
- **价值评级**：🔴 很有用
- **理由**：
  - 四种场景结果类型是解决"场景太平"问题的直接工具
  - Fichtean Curve 特别适合《鸾凤囚》的宫斗/朝堂节奏——不需要漫长铺垫，每个小危机都是钩子
  - Sequel 的 Reaction→Dilemma→Decision 结构能让角色决策更有层次，避免"角色做了个决定"然后直接跳转的机械感
- **与已有知识对比**：增量。已有"章末钩子"和"去人机味三板斧"，但缺少场景级别的结构框架

##### 用在哪？
- **具体场景**：
  1. 写每章前用 Goal-Conflict-Disaster 检查场景是否"有事发生"
  2. 每5-8章安排一个 Fichtean Curve 的 crisis 点，确保张力螺旋上升
  3. 场景间用 Sequel 三步法写过渡段落（情绪→犹豫→抉择）
  4. 朝堂/宫斗场景用 Try-Fail Cycle，让潘玥试两三个方案才解决问题
- **预期改善**：
  - 场景不再"写了一堆但什么都没发生"
  - 角色决策过程更有层次感和可信度
  - 章节间节奏更有呼吸感（紧张→松弛→更紧张）
