### 2026-04-05 Stryker 变异测试实战

#### 学了什么
- **核心洞察**：100% 代码覆盖率 ≠ 测试能捕获 bug，变异测试验证测试质量
- **Stryker 配置**：thresholds(high=80/low=60/break=50), incremental 模式加速 CI
- **报告解读**：Killed(好) / Survived(需改进) / No Coverage(无覆盖) / Timeout(无限循环)

#### 有用吗？
- 价值评级：🔴 高价值
- 理由：直接解决"测试覆盖率高但 bug 仍逃逸"的痛点

#### 用在哪？
- **店小秘核心模块**：
  - 订单计算逻辑（金额/税费/运费）
  - 库存校验逻辑（超卖防护/库存同步）
  - 引入 Stryker 验证测试是否能捕获边界条件 bug

### 2026-04-05 测试策略文档编写

#### 学了什么
- **测试策略定义**："如何测试应用"——明确测试流程/策略/范围/覆盖率
- **测试策略 vs 测试计划**：策略是计划的子集 (IEEE 829-2008)，策略=方法论，计划=策略 + 项目计划
- **核心价值**：消除不确定性，定义测试范围，降低遗漏测试活动的风险
- **关键实践**：策略文档必须与团队讨论，确保方法/责任一致性

#### 有用吗？
- 价值评级：🟡 增量
- 理由：测试策略概念已知，但 IEEE 标准和与测试计划的区分是新增认知

#### 用在哪？
- **店小秘测试流程**：
  - 为新功能/大版本编写测试策略文档
  - 明确测试范围/方法/工具/环境/准入准出标准
  - 与开发/产品团队对齐测试方法
