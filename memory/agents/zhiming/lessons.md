## 🔄 可自动化操作

### 1. Gateway 健康巡检 → 已有脚本，可扩展
- 现状：`workspace-health.sh` 已覆盖基础检查
- **改进**：增加 `openclaw gateway status` + 端口探测 + 日志异常检测（grep ERROR/WARN 最近50行）
- 产出：写健康状态到 `memory/health-state.json`，心跳读取判断是否需要告警

### 2. Cron 400 错误自动修复
- 现状：08晨会/10内容创意/23休息连续400错误，人工排查根因（delivery缺少channel:"feishu"）
- **改进**：写脚本 `scripts/cron-health-check.sh`
  ```bash
  # 扫描所有cron，检测连续失败 >3次的
  # 自动对比成功cron vs 失败cron的配置差异
  # 输出修复建议
  ```
- 规则：**所有 cron delivery 必须包含 channel:"feishu"** → 写入 AGENTS.md

### 3. Memory 文件归档自动化 → 已有，可优化
- 现状：`cleanup-daily.sh` 清理7天前文件
- **改进**：归档前先执行一次"知识提取"——扫描待归档文件，将关键决策/教训提取到 lessons.md 或 MEMORY.md，再归档
- 产出：`scripts/archive-with-extract.sh`

### 4. 会话归档去重
- 现状：每次归档扫描所有活跃会话，可能重复提取
- **改进**：`memory/archive-state.json` 记录每个会话上次归档的 message_id，只提取增量
- 规则：**归档 = 增量，不是全量**

## 📏 可规则化的判断

### 1. 飞书消息发送判断规则
- **规则**：优先用 `message` 系统工具，只在用户明确说"以我身份发"时才用 `feishu_im_user_message`
- **规则**：飞书消息 content 必须是合法 JSON，text 类型用 `{"text":"..."}`, post 用嵌套结构
- 已写入 AGENTS.md，但可补充到 TOOLS.md 作为速查

### 2. 浏览器使用规则
- **规则**：默认 sandbox 浏览器，禁止主动 `profile="user"`
- **规则**：只有主人明确说"用我的浏览器"才用 host 浏览器
- 已在 AGENTS.md 中，但可加到每次浏览器调用前的 checklist

### 3. 搜索降级规则（子Agent）
- **规则链**：web_search → exec+curl API → exec+curl抓HTML → web_fetch → sessions_send求助
- **规则**：搜索工具不可用时禁止直接放弃，必须尝试降级链
- 已在 AGENTS.md 中

### 4. 安全判断规则
- **规则**：`trash` > `rm`（可恢复优于永久删除）
- **规则**：外部操作（发邮件/发推/公开发布）必须先问
- **规则**：Docker 镜像用 digest pinning，禁止裸 tag
- **规则**：Python 依赖禁止 `>=`，必须 `==` 或 lockfile

### 5. Cron delivery 规则（新）
- **规则**：所有 cron delivery payload 必须包含 `channel:"feishu"`
- **规则**：创建 cron 后立即验证首次执行是否成功
- 根因：2026-04-01 6个cron因缺少channel参数连续400

## ⚡ 可优化的流程

### 1. 心跳执行流程 → 拆分优先级
- 现状：心跳每次做一堆检查，token消耗大
- **优化**：
  - **高优**：Gateway状态 + 磁盘告警（<1GB）
  - **中优**：Cron健康 + 新消息归档
  - **低优**：MEMORY.md维护 + lessons.md更新（每3天一次）
- 规则：**心跳只做高优，中优轮值，低优周末做**

### 2. 多Agent协作 → 统一汇报格式
- 现状：各Agent汇报格式不统一，主Agent需要手动整合
- **优化**：定义标准汇报模板
  ```
  ## [Agent名] · 任务摘要
  - **状态**: ✅完成 / ⚠️部分完成 / ❌失败
  - **产出**: 文件路径/关键结论
  - **待跟进**: 需主人决策的事项
  - **耗时**: Xmin
  ```

### 3. Skills 自检 → 统一入口
- 现状：`workspace-health.sh` 和 `cleanup-daily.sh` 各自独立
- **优化**：写 `scripts/self-check.sh` 统一入口
  ```bash
  # 1. Gateway健康
  # 2. 磁盘/内存
  # 3. Cron状态
  # 4. Memory文件大小
  # 5. 输出结构化报告
  ```

### 4. 学习闭环流程 → 强制写入配置
- 现状：学→评→用→反馈，但"用"这步容易跳过
- **优化**：每次学习必须回答3个问题：
  1. 这个知识改了我哪个配置文件？
  2. 这个知识造了什么工具？
  3. 如果都没做，为什么值得记？

### 5. 飞书文档创建流程 → 模板化
- 现状：每次创建飞书文档从零开始
- **优化**：常用文档类型建模板（面试指南/安全审计/技术方案），直接填充
