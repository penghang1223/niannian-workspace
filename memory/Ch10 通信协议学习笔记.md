## 📚 Ch10 通信协议核心内容

### 为什么需要通信协议？

**问题场景**：
```
项目经理 Agent → 玄机 Agent
"帮我写个登录功能"

❌ 没有协议：
- 消息格式不统一
- 任务描述不清晰
- 结果无法自动化处理
- 无法追踪进度

✅ 有协议：
- 标准化消息格式
- 明确的任务定义
- 可自动化的结果处理
- 完整的追踪链路
```

---

## 🔧 三大主流协议

### 1. MCP (Model Context Protocol) 🔴

**定位**：模型上下文协议，让 AI 模型与外部工具/数据源通信

**核心概念**：
```
┌─────────────┐
│   AI Model  │
│  (LLM/Claude)│
└──────┬──────┘
       │ MCP
       ▼
┌─────────────┐
│   Server    │
│ (工具/数据源) │
└─────────────┘
```

**消息格式**：
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "search",
    "arguments": {
      "query": "用户登录功能"
    }
  }
}
```

**适用场景**：
- ✅ AI 模型调用外部工具
- ✅ 访问数据库/API
- ✅ 读取文件系统

---

### 2. A2A (Agent to Agent) 🟡

**定位**：Agent 之间的直接通信协议

**核心概念**：
```
Agent A → Agent B
   ↓         ↓
消息队列   消息队列
   ↓         ↓
Agent B ← Agent A
```

**消息格式**：
```json
{
  "message_id": "msg_001",
  "sender": "project_manager",
  "receiver": "developer",
  "type": "task_assignment",
  "content": {
    "task": "实现用户登录 API",
    "requirements": [
      "支持邮箱/密码登录",
      "支持 OAuth2.0",
      "返回 JWT token"
    ],
    "deadline": "2026-03-03 12:00",
    "priority": "high"
  },
  "timestamp": "2026-03-02T22:45:00Z"
}
```

**适用场景**：
- ✅ 多 Agent 协同
- ✅ 任务分配和结果汇报
- ✅ Agent 间的知识共享

---

### 3. ANP (Agent Network Protocol) 🟢

**定位**：Agent 网络协议，支持大规模 Agent 集群通信

**核心概念**：
```
       ┌──────────┐
       │  Gateway │
       └────┬─────┘
            │
    ┌───────┼───────┐
    │       │       │
┌───┴───┐ ┌─┴─┐ ┌──┴──┐
│Agent1 │ │...│ │AgentN│
└───────┘ └───┘ └─────┘
```

**特点**：
- ✅ 支持大规模 Agent 集群
- ✅ 服务发现机制
- ✅ 负载均衡
- ✅ 故障恢复

## 🎯 主人的多 Agent 团队应该用什么？

### 分析主人的配置

| Agent | 角色 | 通信需求 |
|-------|------|----------|
| **项目经理** | 协调器 | 分配任务、收集结果 |
| **玄机** | 执行者 | 接收任务、提交代码 |
| **年年** | 智能助手 | 质量评估、意图预测 |

### 推荐方案：**A2A + 自定义扩展** ⭐

**原因**：
1. ✅ Agent 数量少（4 个），不需要 ANP 的复杂性
2. ✅ 需要任务分配和结果汇报（A2A 的核心功能）
3. ✅ 可以自定义扩展质量评估和意图预测

### 1. 消息格式

```python
class AgentMessage:
    """Agent 通信消息格式"""
    
    # 消息元数据
    message_id: str          # 唯一 ID (UUID)
    conversation_id: str     # 会话 ID (追踪完整流程)
    timestamp: str           # 时间戳 (ISO8601)
    
    # 发送者和接收者
    sender: str              # 发送者 (project_manager/developer/tester/nannan)
    receiver: str            # 接收者 (可以是多个)
    receiver_type: str       # 接收者类型 (agent/human)
    
    # 消息类型
    message_type: str        # 消息类型
    priority: int            # 优先级 (1-5, 5 最高)
    
    # 消息内容
    content: dict            # 具体内容 (根据类型变化)
    
    # 状态追踪
    status: str              # 状态 (pending/sent/delivered/read/replied)
    parent_message_id: str   # 父消息 ID (如果是回复)
    
    # 扩展字段
    metadata: dict           # 元数据 (用于质量评估等)
```

## 🏗️ 实现架构

### 消息队列设计

```python
class MessageQueue:
    """Agent 消息队列"""
    
    def __init__(self):
        self.queues = {
            "project_manager": [],
            "developer": [],
            "tester": [],
            "nannan": []
        }
    
    def send(self, message: AgentMessage):
        """发送消息到接收者队列"""
        receiver = message.receiver
        self.queues[receiver].append(message)
    
    def receive(self, agent_id: str) -> List[AgentMessage]:
        """接收者获取消息"""
        messages = self.queues[agent_id]
        self.queues[agent_id] = []  # 清空队列
        return messages
    
    def status(self, agent_id: str) -> dict:
        """获取队列状态"""
        return {
            "pending_messages": len(self.queues[agent_id]),
            "agent_id": agent_id
        }
```

### 协调器实现框架

```python
class ProjectManagerAgent:
    """项目经理 Agent (协调器)"""
    
    def __init__(self):
        self.message_queue = MessageQueue()
        self.conversations = {}  # 追踪会话
        self.nannan = NannanAssistant()  # 年年助手
    
    def receive_request(self, human_request: str):
        """接收主人请求"""
        # 1. 年年意图预测
        prediction = self.nannan.predict_intent(human_request)
        
        # 2. 任务分解
        tasks = self.decompose_tasks(prediction)
        
        # 3. 分配任务
        for task in tasks:
            message = self.create_task_message(task)
            self.message_queue.send(message)
        
        # 4. 追踪进度
        self.track_progress(tasks)
    
    def collect_results(self, conversation_id: str):
        """收集结果"""
        results = []
        for message in self.message_queue.receive("project_manager"):
            if message.message_type == "task_submit":
                results.append(message)
        
        # 年年质量评估
        for result in results:
            quality = self.nannan.evaluate(result)
            result.metadata["quality"] = quality
        
        # 整合结果
        return self.integrate_results(results)
```
