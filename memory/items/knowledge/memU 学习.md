## 🌟 核心特性

### 1. 文件系统式记忆
```
文件系统          memU 记忆
├── 📁 Folders    →  🏷️ Categories (自动组织主题)
├── 📄 Files      →  🧠 Memory Items (提取事实、偏好、技能)
├── 🔗 Symlinks   →  🔄 Cross-references (记忆互相关联)
└── 📂 Mount pts  →  📥 Resources (对话、文档、图片)
```

### 2. 三层架构
| 层级 | 作用 | 检索方式 |
|------|------|----------|
| **Resource** | 原始数据 | 直接访问 |
| **Item** | 原子记忆 | 精准检索 |
| **Category** | 主题分类 | 概览浏览 |

### 3. 主动记忆循环
```
用户输入 → Agent 处理 → 回复用户
    ↓         ↓           ↓
memU 监控 ← 提取记忆 ← 预测下一步
    ↓
主动任务（预取上下文、推荐内容）
```

### 4. 双模式检索
| 模式 | 速度 | 成本 | 适用场景 |
|------|------|------|----------|
| **RAG** | 毫秒级 | embedding only | 实时建议 |
| **LLM** | 秒级 | LLM 推理 | 复杂预测 |

## 🎯 核心优势

### 1. 成本优化
- 缓存洞察，避免重复分析
- 分层检索，减少 LLM 调用
- 批量处理，降低 API 成本

### 2. 主动预测
- 从交互中自动提取偏好
- 预测用户下一步需求
- 提前准备上下文

### 3. 结构化组织
- 像文件系统一样好导航
- 记忆间自动链接
- 持久化可导出
