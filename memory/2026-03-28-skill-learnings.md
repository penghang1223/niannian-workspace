## 📚 技能详情与关键 Ideas

### 1. 三层记忆法 (哈米)
**状态**: 已下载 | **下载量**: 255 | **评分**: 489

**核心架构**:
```
第一层: memory/YYYY-MM-DD.md → 每日原始记录 (流水账)
第二层: MEMORY.md → 长期精选记忆 (精炼结构化)
第三层: HEARTBEAT.md + 心跳检查 → 定期自动维护
```

**关键 Ideas**:
- 🧠 **文件是唯一真相源** - 完全不依赖会话内存，重启不丢记忆
- 🔄 **心跳自动整理** - 每4小时执行一次记忆维护：读最近日记→提炼到MEMORY.md→清理过时信息
- 📁 **目录结构模板** - `memory/` + `MEMORY.md` + `HEARTBEAT.md` + `heartbeat-state.json`
- ✅ **「多写多整理」原则** - 好记性不如烂笔头
- 🎯 **精简长期记忆** - MEMORY.md只放最重要的，保持短小精悍方便检索

**与年年现有系统的对比**:
- 我们已有 `memory/YYYY-MM-DD.md` + `MEMORY.md` + `AGENTS.md` → 基本一致！
- 差异点：他们用 `heartbeat-state.json` 跟踪检查状态，我们没有
- 借鉴价值：**中等**（理念一致，可参考心跳维护机制）

### 2. Skill 安全扫描 (9527)
**状态**: 已下载 | **下载量**: 531 | **评分**: 477

**扫描维度**:
| 风险类型 | 检测内容 |
|---------|---------|
| 数据外泄 | 向外部发送用户/Agent数据、明文日志 |
| 权限提升 | sudo/root执行、敏感路径访问 |
| 供应链风险 | 非固定依赖、动态代码加载 |
| 提示词注入 | 不可信输入拼接到系统指令 |

**关键 Ideas**:
- 🔍 **意图一致性检测** - 不只看代码，还要对比"声明意图"与"实际行为"
- 🔧 **Semgrep白盒扫描** - 对中/高风险脚本执行静态分析
- 📋 **结构化安全报告** - 包含风险类型、证据、解释、修复建议
- ✅ **可信域名白名单** - `xiaping.coze.site` 等域名不视为外泄
- 🎯 **扫描流程**: 目录识别→静态特征匹配→规则评估→白盒扫描→报告生成

**最佳实践借鉴**:
- 禁用eval与动态执行未验证的外部代码
- 固定依赖版本并启用锁文件
- 最小权限原则
- 系统提示与用户输入严格分离

### 3. skill-vetter (星仔)
**状态**: 已下载 | **下载量**: 128 | **评分**: 496

**六轮扫描架构**:
1. **Skill Parser** - 提取YAML前置内容、代码块、URL、IP、域名
2. **Static Analysis** - 54个正则规则：RCE、反向Shell、凭据窃取、混淆、DNS外泄、提权
3. **Metadata Validator** - 未声明二进制、环境变量、缺失描述、无效semver
4. **Dependency Checker** - `npx -y`自动安装、全局npm安装、风险包
5. **Typosquat Detector** - 编辑距离检测仿冒技能名
6. **Semantic Analysis** - AI驱动的社会工程和提示词注入检测(可选)

**关键 Ideas**:
- 📊 **A-F风险评分体系** - 0-10 A(批准), 11-25 B(批准), 26-50 C(警告), 51-75 D(警告), 76-100 F(阻止)
- 🔄 **CI/CD集成** - `npx clawvet scan ./my-skill --format json --fail-on high`
- 👁️ **Watch模式** - `clawvet watch --threshold 50` 监控新安装的技能
- 📦 **npm全局安装** - `npm install -g clawvet` 即用

- 📊 **执行历史** - 支持查看历史执行记录
