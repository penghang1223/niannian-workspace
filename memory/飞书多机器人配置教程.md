## 3. 配置机器人

### 3.1 启用机器人功能

对**每个应用**执行以下操作：

1. 进入应用详情页
2. 左侧菜单：**机器人** → **创建机器人**
3. 填写机器人信息：
   ```
   机器人名称：项目经理（与角色一致）
   机器人头像：🤖
   功能描述：负责任务分配和结果整合
   ```
4. 点击**创建**

### 3.2 配置机器人功能

创建机器人后，配置：

- ✅ **支持单聊** - 允许私聊
- ✅ **支持群聊** - 允许群消息
- ✅ **支持@机器人** - 允许@触发
- ✅ **支持消息卡片** - 发送富文本消息

### 3.3 设置机器人回复模式

选择：**Webhook**（推荐）或**事件订阅**

**Webhook 模式**：
- 简单，适合单向通知
- ❌ 不适合多 Agent 协同

**事件订阅模式**：
- 强大，支持双向通信
- ✅ **推荐用于多 Agent**

## 4. 配置事件订阅

### 4.1 准备回调 URL

#### 开发环境（本地测试）

使用 ngrok 暴露本地服务：

```bash
# 1. 安装 ngrok
brew install ngrok

# 2. 启动本地服务（假设端口 8080）

# 3. 在另一个终端启动 ngrok
ngrok http 8080

# 4. 获取临时域名（类似）
https://abc123.ngrok.io
```

#### 生产环境（服务器部署）

使用真实域名：
```
https://your-domain.com/feishu/webhook
```

### 4.2 配置事件订阅 URL

对**每个应用**：

1. 进入应用详情页
2. 左侧菜单：**事件订阅**
3. 配置：
   ```
   订阅地址：https://abc123.ngrok.io/feishu/webhook
   加密密钥：（自动生成，复制保存）
   ```
4. 点击**保存**

### 4.3 订阅事件类型

对**每个应用**，订阅以下事件：

| 事件类型 | 说明 | 必需 |
|----------|------|------|
| `im.message.receive_v1` | 接收消息事件 | ✅ |
| `im.chat.member.bot.added_v1` | 机器人被拉入群 | ✅ |
| `im.chat.update` | 群聊更新 | ⚠️ 可选 |

**配置步骤**：
1. 点击**添加事件**
2. 搜索事件名称
3. 选择事件版本（选 v1）
4. 点击**确定**

### 4.4 验证订阅地址

飞书会发送验证请求，确保后端正确响应：

```python

def get_tenant_access_token(app_id: str, app_secret: str) -> str:
    """获取飞书访问令牌"""
    url = "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal"
    
    payload = {
        "app_id": app_id,
        "app_secret": app_secret
    }
    
    response = requests.post(url, json=payload)
    data = response.json()
    
    if data.get('code') == 0:
        return data.get('tenant_access_token')
    else:
        raise Exception(f"获取 token 失败：{data}")


def send_message(chat_id: str, content: dict, app_id: str, app_secret: str):
    """发送消息到飞书"""
    token = get_tenant_access_token(app_id, app_secret)
    
    url = "https://open.feishu.cn/open-apis/im/v1/messages"
    
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }
    
    payload = {
        "receive_id": chat_id,
        "msg_type": "text",
        "content": json.dumps(content)
    }
    
    response = requests.post(url, headers=headers, json=payload)
    data = response.json()
    
    if data.get('code') == 0:
        print(f"✅ 消息已发送：{chat_id}")
        return data
    else:
        print(f"❌ 发送失败：{data}")
        return None

    if data.get('type') == 'message':

    # 3. 处理其他事件
    return jsonify({'status': 'ok'})


def handle_message_event(data: dict):
    """处理消息事件"""
    try:
        # 解析消息
        header = data.get('header', {})
        event = data.get('event', {})
        
        message = event.get('message', {})
        sender = event.get('sender', {})
        
        chat_id = message.get('chat_id')
        content = json.loads(message.get('content', '{}'))
        text = content.get('text', '')
        mentions = content.get('mentions', [])
        
        sender_id = sender.get('sender_id', {}).get('open_id')
        sender_type = sender.get('sender_type', 'user')
        
        print(f"\n📨 收到消息:")
        print(f"   群聊：{chat_id}")
        print(f"   发送者：{sender_id}")
        print(f"   内容：{text[:50]}...")
        print(f"   @提及：{mentions}")
        
        # 检查是否有@机器人
        if mentions:
            for mention in mentions:
                mention_type = mention.get('type')  # user, bot, etc.
                mention_id = mention.get('id')      # 被@的用户/机器人 ID
                
                print(f"   被@类型：{mention_type}, ID: {mention_id}")
                
                # 根据@的机器人调用对应的 Agent
                if mention_type == 'bot':
                    return handle_bot_mention(mention_id, chat_id, text, sender_id)
        
        return jsonify({'status': 'ok'})
    
    except Exception as e:
        print(f"❌ 处理消息失败：{e}")
        return jsonify({'status': 'error', 'message': str(e)})


def handle_bot_mention(bot_id: str, chat_id: str, text: str, sender_id: str):
    """处理@机器人"""
    print(f"\n🤖 机器人 {bot_id} 被@")
    
    # 识别是哪个 Agent
    agent_name = None
    for name, config in APPS.items():
        if config.get('app_id') == bot_id:
            agent_name = name
            break
    
    if not agent_name:
        print(f"⚠️  未知机器人：{bot_id}")
        return jsonify({'status': 'ok'})
    
    print(f"✅ 识别为 Agent: {agent_name}")
    
    # 根据 Agent 类型处理
    if agent_name == 'project_manager':
        # 项目经理：接收主人请求
        conversation_id = pm.receive_human_request(text, sender_id)
        active_sessions[conversation_id] = {
            'chat_id': chat_id,
            'sender_id': sender_id,
            'agent': 'project_manager'
        }
        
        # 年年进行意图预测
        nannan.process_requests()
        
        # 获取年年的建议
        nannan_messages = queue.receive('nannan')
        for msg in nannan_messages:
            if msg.message_type == 'intent_prediction':
                # 根据建议分配任务
                tasks = extract_tasks_from_prediction(msg.content)
                pm.assign_tasks(conversation_id, tasks)
        
        # 开发和测试 Agent 处理任务
        dev.process_tasks()
        tester.process_tasks()
        
        # 收集结果
        results = pm.collect_results(conversation_id)
        summary = pm.integrate_results(results)
        
        # 回复到群里
        app_config = APPS['project_manager']
        send_message(
            chat_id=chat_id,
            content={"text": summary},
            app_id=app_config['app_id'],
            app_secret=app_config['app_secret']
        )
    
    elif agent_name == 'nannan':
        # 年年：直接回复
        nannan_response = generate_nannan_response(text)
        app_config = APPS['nannan']
        send_message(
            chat_id=chat_id,
            content={"text": nannan_response},
            app_id=app_config['app_id'],
            app_secret=app_config['app_secret']
        )
    
    return jsonify({'status': 'ok'})
```

## 9. 故障排查

### 问题 1：@机器人没有反应

**可能原因**：
- ❌ 事件订阅未配置
- ❌ 后端服务未启动
- ❌ 权限未配置

**解决方法**：
1. 检查飞书应用后台 → 事件订阅 → 订阅地址是否正确
2. 检查后端服务是否运行：`curl http://localhost:8080/health`
3. 检查权限配置 → 权限管理 → 是否已申请必要权限

### 问题 2：收不到消息

**可能原因**：
- ❌ 机器人不在群里
- ❌ 消息类型不支持

**解决方法**：
1. 检查群成员列表，确认机器人已加入
2. 检查事件订阅 → 是否订阅了 `im.message.receive_v1`

### 问题 3：发送消息失败

**可能原因**：
- ❌ App Secret 错误
- ❌ 权限不足

**解决方法**：
1. 检查 config.py 中的 App Secret 是否正确
2. 检查权限管理 → 是否有 `im:message.send_as_bot` 权限

### 问题 4：ngrok 域名变化

**问题**：每次重启 ngrok 域名都会变化

**解决方法**：
1. 使用 ngrok 付费版（固定域名）
2. 或部署到服务器（推荐）
3. 或每次重启后更新飞书后台配置

### 开发环境 vs 生产环境

| 环境 | 推荐方案 | 成本 |
|------|----------|------|
| **开发** | 本地 + ngrok | 免费 |
| **测试** | 云服务器 | ¥50-100/月 |
| **生产** | 云服务器 + 域名 | ¥200-500/月 |

### 安全建议

1. **不要公开 App Secret**
2. **使用 HTTPS**（生产环境）
3. **验证请求签名**（防止伪造）
4. **限制访问频率**（防止滥用）

### 性能优化

1. **使用异步处理**（asyncio）
2. **添加消息队列**（Redis/RabbitMQ）
3. **缓存 Access Token**
4. **数据库持久化**
