## 一、系统概述

### 1.1 核心理念

让 AI 在主人睡眠期间自主完成小型应用的构建，第二天醒来即可验收成果。

```
┌─────────────────────────────────────────────────────────────┐
│                    隔夜构建流程                              │
├─────────────────────────────────────────────────────────────┤
│  主人提交想法 → AI 生成任务 → 夜间执行 → 晨间验收          │
│      ↓           ↓          ↓          ↓                    │
│   22:00       22:30      23:00-6:00    7:00                │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

- **自主性**：AI 可独立完成任务执行，无需实时干预
- **安全性**：所有操作可追溯、可回滚
- **渐进性**：每天 4-5 个任务，避免过度构建
- **透明性**：主人随时可查看进度和日志

## 三、任务生成逻辑

### 3.1 每日任务生成规则

**时间触发：** 每天 22:30 自动生成当日任务

**任务数量：** 4-5 个/天（根据复杂度动态调整）

**任务类型分布：**
| 类型 | 数量 | 说明 |
|------|------|------|
| 核心功能 | 1-2 个 | 当天最重要的功能实现 |
| 辅助功能 | 1-2 个 | 配套功能或优化 |
| 测试/文档 | 1 个 | 单元测试或文档编写 |
| 技术债务 | 0-1 个 | 重构或优化 |

### 3.2 任务生成算法

```javascript
// 伪代码示例
function generateDailyTasks(appIdea, currentProgress) {
  const tasks = [];
  
  // 1. 分析当前进度
  const completedFeatures = analyzeProgress(currentProgress);
  
  // 2. 确定下一步优先级
  const nextFeatures = prioritizeFeatures(appIdea, completedFeatures);
  
  // 3. 生成任务（4-5 个）
  tasks.push(createCoreTask(nextFeatures[0]));  // 核心功能
  tasks.push(createCoreTask(nextFeatures[1]));  // 核心功能
  tasks.push(createAuxiliaryTask(nextFeatures[2]));  // 辅助功能
  tasks.push(createTestTask(nextFeatures[0]));  // 测试
  
  // 4. 如果有剩余时间，添加技术债务任务
  if (estimatedTime < 6) {
    tasks.push(createDebtTask());
  }
  
  return tasks;
}
```

### 3.3 任务复杂度评估

| 复杂度 | 预估时间 | 每日数量 |
|--------|----------|----------|
| 简单 | 30-60 分钟 | 2-3 个 |
| 中等 | 60-90 分钟 | 1-2 个 |
| 复杂 | 90-120 分钟 | 0-1 个 |

## 五、避免竞态条件的文件设计

### 5.1 核心问题

在多 Agent 协作或长时间运行任务时，可能出现：
- 任务重复执行
- 状态更新冲突
- 日志写入混乱

### 5.2 文件锁机制

**AUTONOMOUS.md - 自主执行状态文件**

```markdown
# AUTONOMOUS.md - 自主执行状态

**最后更新时间：** 2026-03-19T22:30:00+08:00
**执行 Agent：** dev_engineer
**执行状态：** running | idle | paused | error

## 当前执行任务

| 任务 ID | 开始时间 | 预计完成 | 状态 |
|---------|----------|----------|------|
| TASK-20260319-001 | 22:30 | 23:30 | running |
| TASK-20260319-002 | - | - | pending |

## 执行锁

- **锁持有者：** dev_engineer
- **锁获取时间：** 2026-03-19T22:30:00+08:00
- **锁过期时间：** 2026-03-20T06:00:00+08:00
- **心跳最后更新：** 2026-03-19T22:35:00+08:00

## 安全机制

1. 同一时间只有一个 Agent 持有执行锁
2. 锁超时后自动释放（防止死锁）
3. 任务执行前必须获取锁
4. 任务完成后必须释放锁
```

**获取锁流程：**
```bash
# 伪代码
function acquireLock(agentId) {
  const lock = readFileSync('AUTONOMOUS.md');
  
  // 检查锁是否过期
  if (lock.expiresAt < now()) {
    // 锁已过期，可以获取
    return acquireNewLock(agentId);
  }
  
  // 锁未过期，检查是否是自己持有的
  if (lock.holder === agentId) {
    // 刷新锁
    return refreshLock(agentId);
  }
  
  // 锁被其他 Agent 持有
  throw new Error('Lock held by another agent');
}
```

### 5.4 日志写入规范

**原子写入原则：**
```javascript
// 使用临时文件 + 原子重命名
function appendLog(entry) {
  const tempFile = `tasks-log.md.tmp.${Date.now()}`;
  const logFile = 'tasks-log.md';
  
  // 1. 读取现有内容
  const content = readFileSync(logFile);
  
  // 2. 追加新日志
  const newContent = content + '\n' + entry;
  
  // 3. 写入临时文件
  writeFileSync(tempFile, newContent);
  
  // 4. 原子重命名
  renameSync(tempFile, logFile);
}
```

**日志格式规范：**
- 每个任务独立章节
- 包含完整时间线
- 记录所有输出文件
- 关联 Git Commit
