## 🎯 各 Agent 职责

### 1️⃣ 研究 Agent（岁岁）

**职责**：
- 扫描热点话题（小红书/抖音/微博）
- 监控竞品内容
- 追踪行业趋势
- 生成内容机会清单

**工作时间**：每日 10:00 AM

**输出**：
- 3-5 个内容创意
- 热点话题列表
- 竞品分析摘要

**工具**：
- web_search（热点搜索）
- web_fetch（内容抓取）
- 记忆系统（趋势分析）

---

### 2️⃣ 写作 Agent（娜尔）

**职责**：
- 根据研究结果撰写内容
- 生成脚本/文案/文章
- 优化内容结构和语气
- 添加合适的标签和关键词

**工作时间**：每日 11:00 AM（接收研究结果后）

**输出**：
- 完整内容草稿
- 标题建议（3-5 个）
- 标签建议

**工具**：
- 内容模板库
- SEO 关键词工具
- 语气优化提示

---

### 3️⃣ 设计 Agent（夕尔）

**职责**：
- 根据内容生成配图
- 设计封面/缩略图
- 优化视觉效果
- 确保品牌一致性

**工作时间**：每日 14:00 PM（接收写作结果后）

**.output**：
- 封面图（1080x1920）
- 配图（多张）
- 视觉元素

**工具**：
- 图像生成 API
- 设计模板
- 品牌规范

## 📱 发布渠道

### 小红书
- 格式：图文笔记
- 尺寸：1080x1920（竖版）
- 标签：5-10 个
- 发布时间：18:00-20:00

### 抖音
- 格式：短视频
- 时长：15-60 秒
- 标签：3-5 个
- 发布时间：19:00-21:00

### 微信公众号
- 格式：长文章
- 配图：3-5 张
- 发布时间：20:00-22:00

## 🔧 技术实现

### 方案 1：飞书多维表格（推荐）

**优势**：
- 可视化任务管理
- 自动状态流转
- 团队协作方便

**实现**：
1. 创建内容工厂多维表格
2. 定义字段：标题、状态、负责人、截止时间等
3. 设置自动化规则
4. 各 Agent 更新对应状态

---

### 方案 2：文件驱动

**文件结构**：
```
content-factory/
├── research/        # 研究结果
├── scripts/         # 内容草稿
├── thumbnails/      # 设计稿
├── review/          # 审核中
├── approved/        # 已批准
└── published/       # 已发布
```

**流程**：
1. 岁岁创建 `research/YYYY-MM-DD-热点.md`
2. 娜尔读取并创建 `scripts/YYYY-MM-DD-内容.md`
3. 夕尔读取并创建 `thumbnails/YYYY-MM-DD-设计/`
4. 年年审核后移动到 `approved/`
5. 岁岁发布后移动到 `published/`
