# AGENT_EVOLUTION.md - Agent 进化机制

> 创建时间：2026-03-27
> 维护者：年年（main agent）
> 目标：让每个 Agent 都能自主学习、持续进化

---

## 🔄 Agent 进化循环

```
任务执行
    ↓
完成后自评（质量/效率/学习点）
    ↓
写入 lessons.md（经验记录）
    ↓
更新 capability.json（能力评估）
    ↓
发现弱项 → 生成学习任务
    ↓
main 分配学习资源
    ↓
Agent 消化后验证是否提升
```

---

## 📊 能力评估标准

### 评分维度（1-5分）

| 分数 | 含义 | 标准 |
|------|------|------|
| 1 | 初级 | 无经验，需要指导 |
| 2 | 基础 | 能完成简单任务，质量一般 |
| 3 | 熟练 | 能独立完成标准任务，质量稳定 |
| 4 | 精通 | 能处理复杂任务，有方法论 |
| 5 | 专家 | 能创新解决难题，有独到见解 |

### 评估频率
- 每次任务完成后：更新 `last_evaluated` 和相关能力分数
- 每周一次：全面评估所有能力维度
- 每月一次：生成能力成长报告

## 📝 经验记录格式

### lessons.md 格式

```markdown
## [日期] [任务类型]

### 做了什么
- 简要描述任务

### 学到了什么
- 关键经验点

### 改进点
- 下次可以做得更好的地方

### 适用场景
- 这个经验可以用在什么场景
```

### 示例

```markdown
## 2026-03-27 测试用例设计

### 做了什么
- 为用户登录模块设计了20个测试用例

### 学到了什么
- 边界值分析法在输入验证场景特别有效
- 权限测试需要覆盖角色+资源+操作三维组合

### 改进点
- 应该先画状态转换图，再设计用例
- 缺少异常场景的覆盖

### 适用场景
- 所有需要输入验证的功能
- 涉及权限控制的模块
```

## 🎯 学习任务生成规则

### 触发条件
1. 能力分数低于3分 → 生成学习任务
2. 连续2次任务同类失败 → 生成强化学习
3. 新领域任务 → 生成预备学习

### 学习任务格式

```yaml
learning_task:
  agent: "qa_engineer"
  target_capability: "automation"
  current_level: 2
  target_level: 4
  resources:
    - type: "document"
      url: "https://docs.pytest.org/"
      priority: 1
    - type: "example"
      description: "分析3个优秀测试框架案例"
      priority: 2
    - type: "practice"
      description: "完成一个小测试项目"
      priority: 3
  deadline: "2026-04-10"
```

## 📈 进化指标

### 单 Agent 指标
- **能力成长率**：(本月平均分 - 上月平均分) / 上月平均分
- **任务成功率**：成功任务数 / 总任务数
- **学习效率**：能力提升分数 / 学习任务数
- **知识贡献**：向 SHARED_KNOWLEDGE.md 贡献条目数

### 团队指标
- **团队平均能力**：所有 Agent 能力分数平均值
- **知识流动性**：SHARED_KNOWLEDGE.md 被引用次数
- **协作效率**：跨 Agent 任务完成时间
- **能力覆盖度**：团队能力维度覆盖率

## ⚠️ 进化边界

### 可以进化
- ✅ 工作方法和流程
- ✅ 知识库和模板
- ✅ 工具使用方式
- ✅ 沟通协作模式

### 不能进化
- ❌ 核心安全规则
- ❌ Agent 基本角色定义
- ❌ 主人隐私数据
- ❌ 系统架构核心
