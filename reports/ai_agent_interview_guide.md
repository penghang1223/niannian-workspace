## 🎯 岗位一：AI Agent开发工程师（18-45K）

### 岗位职责

1. **负责设计、构建和优化AI智能体及其应用产品**，快速响应产品业务需求
2. **设计和实现基于LLM的智能体架构**，包括：
   - 任务规划
   - 对话管理
   - 意图识别
   - 知识库
   - 流程工程
3. **开发和维护智能体的后端服务**，确保系统的稳定性和可扩展性
4. **推动AI Agents在架构和性能上的持续优化**
5. **参与和主导大模型智能客服系统的设计和开发**，提升客户交互体验和系统响应效率
6. **对智能体进行评测和调优**

### 任职要求

#### 🔴 核心技能（必须）
- **编程语言**：精通Python、Java、Go其中一种或多种，具备良好的软件框架和应用设计能力
- **Agent框架**：深入了解AI Agents技术栈，掌握LangChain/Llamaindex等LLM开发工具框架
- **Prompt Engineering**：有丰富的智能体开发和Prompt Engineering经验
- **RAG技术**：熟悉LangChain等开发框架及SFT/RAG/Embedding等技术
- **Agent理解**：了解Agent概念，有大模型本地部署及调优经验
- **模型增强**：熟悉模型增强技术(SFT/向量召回/RLHF/Prompts等)

#### 🟡 加分技能
- **机器学习**：熟悉传统机器学习的概念，熟悉机器学习各种任务的定义、目标、解决的问题、及衡量指标
- **深度学习框架**：熟悉TensorFlow、PyTorch等，可以快速搭建和训练模型
- **数学基础**：对概率论、线性代数、微积分了解，能够通过数学方法分析模型的性能和误差

## 📚 面试知识点详解

### 1. Agent架构设计

**必考问题**：
- 如何设计一个Agent系统？
- Agent的核心组件有哪些？
- 如何实现Agent的自主决策？

**标准答案框架**：
```
Agent核心组件：
1. 感知层 - 接收用户输入和环境信息
2. 规划层 - 任务拆解和步骤规划
3. 记忆层 - 短期记忆和长期记忆
4. 工具层 - 调用外部工具和API
5. 执行层 - 执行具体操作
6. 反思层 - 评估执行结果并调整

设计原则：
- 模块化设计
- 安全优先
- 可观测性
- 可扩展性
```

### 2. RAG系统设计

**必考问题**：
- RAG的完整流程是什么？
- 如何优化RAG的检索质量？
- 如何处理长文档？

**标准答案框架**：
```
RAG完整流程：
1. 文档加载 → 支持多种格式
2. 文档切分 → 合理的chunk大小和重叠
3. 向量化 → 选择合适的Embedding模型
4. 索引存储 → 向量数据库存储
5. 查询向量化 → 用户问题向量化
6. 相似度检索 → 找到相关文档
7. 上下文组装 → 组装Prompt
8. LLM生成 → 生成回答

优化策略：
- 混合检索（向量+关键词）
- Rerank重排序
- 查询改写
- 多轮检索
```

### 3. Multi-Agent设计

**必考问题**：
- 如何设计多Agent系统？
- Agent间如何通信？
- 如何处理Agent间的冲突？

**标准答案框架**：
```
多Agent架构模式：
1. 中心化 - 一个协调者分配任务
2. 去中心化 - Agent间直接通信
3. 层级化 - 多层级Agent结构

通信机制：
- 消息队列
- 共享状态
- API调用
- 事件驱动

冲突解决：
- 优先级机制
- 仲裁Agent
- 投票机制
```

### 4. Prompt Engineering

**必考问题**：
- 如何设计高质量的Prompt？
- 常见的Prompt技术有哪些？
- 如何处理Prompt注入？

**标准答案框架**：
```
Prompt设计原则：
1. 清晰明确 - 避免歧义
2. 结构化 - 使用分隔符和格式
3. 示例驱动 - 提供Few-shot示例
4. 迭代优化 - 根据结果调整

常见技术：
- Zero-shot：直接提问
- Few-shot：提供示例
- CoT：思维链推理
- ReAct：推理+行动

安全防护：
- 输入验证
- 输出过滤
- 权限控制
```

### 5. 工具调用与Function Calling

**必考问题**：
- Function Calling的工作原理？
- 如何设计工具系统？
- 如何处理工具调用失败？

**标准答案框架**：
```
Function Calling流程：
1. 定义工具Schema
2. LLM识别需要调用的工具
3. 提取参数
4. 执行工具
5. 返回结果给LLM
6. LLM生成最终回答

工具设计原则：
- 幂等性设计
- 错误处理
- 超时机制
- 结果验证
```

## 💡 面试技巧

### 回答问题的框架

```
STAR法则：
- Situation（情境）：项目背景
- Task（任务）：你的职责
- Action（行动）：你做了什么
- Result（结果）：取得了什么成果
```

### 技术问题的回答框架

```
1. 是什么 - 概念解释
2. 为什么 - 为什么需要这个技术
3. 怎么做 - 具体实现方式
4. 举例说明 - 实际项目中的应用
```

### 项目经验的表达

```
1. 项目背景 - 为什么做这个项目
2. 技术方案 - 用了什么技术
3. 我的角色 - 我负责什么
4. 遇到的问题 - 遇到了什么挑战
5. 解决方案 - 如何解决的
6. 最终成果 - 取得了什么结果（用数据说话）
```
