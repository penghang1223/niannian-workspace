## 📌 使用说明

### 谁读？
- 所有 Agent 每次任务前读取本文件
- 按角色过滤，只读与自己相关的部分

### 谁写？
- 所有 Agent 完成任务后可写入经验
- 格式：`[Agent名] [日期] 知识内容`

### 冲突处理？
- 以 main agent 最终决定为准
- 有争议的知识标注 `⚠️ 待确认`

### 过期规则？
- 超过30天未被引用 → 标记 `💤 可能过期`
- 超过90天未被引用 → 移入 `archive/`

- **来源**：天工 | 优先级：🔴 高

## 🔴 Node.js Piscina 线程池（2026-03-29）
- **核心能力**：Worker Threads 的生产级封装，支持取消、背压、多函数导出、利用率监控

## 🔴 Agent 任务分解与规划（2026-03-29）
- **规划（Planning）**：预见式，看多步，先做全局规划再执行
- **ReAct**：反应式，看一步走一步，灵活但可能冗余
- **关系**：规划提供恢复脚手架，ReAct 负责执行细节
- **应用**：复杂任务先全局规划（类似 GSD 波次拆解）再执行，减少冗余操作
- **适用**：年年（协调）、望舒（需求拆解）| 优先级：🔴 高

## 🔴 Tailscale K8s Operator 零信任网络（2026-03-29）
- **部署**：Helm 安装 Tailscale Operator，一条注解即可暴露 Service
- **替代方案**：Service 注解暴露替代 LoadBalancer，ACL 标签做端口级权限控制
- **优势**：零公网暴露（零信任）、无证书管理、ACL 精细到端口
- **应用**：搭建 K8s 集群时替代传统 VPN/Ingress
- **适用**：年年、玄机 | 优先级：🔴 高

## 🔴 Docker Compose 多文件合并规则（2026-03-30）
- **三类合并行为**：

- **来源**: 学习汇报 | 2026-04-04
- **核心知识**:
  - **EXPLAIN 树状 plan**: 可视化查询执行计划
  - **成本模型**: `disk*seq_page_cost + rows*cpu_tuple_cost`
  - **Index Scan vs Bitmap Scan**: 选择性 <1% → Index, 1-10% → Bitmap, >10% → Seq
  - **Filter vs Index Cond**: 索引过滤（优）vs 全表后过滤（劣）
- **Plan Nodes**:
  - Seq Scan（全表扫描）- 高成本
  - Index Scan（索引扫描）- 低成本
  - Bitmap Scan（位图扫描）- 中成本
  - Nested Loop/Hash Join/Merge Join（连接方式）
- **应用场景**:
  - 慢查询诊断（EXPLAIN ANALYZE BUFFERS）
