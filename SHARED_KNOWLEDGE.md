## 📌 使用说明

### 谁读？
- 所有 Agent 每次任务前读取本文件
- 按角色过滤，只读与自己相关的部分

### 谁写？
- 所有 Agent 完成任务后可写入经验
- 格式：`[Agent名] [日期] 知识内容`

### 冲突处理？
- 以 main agent 最终决定为准
- 有争议的知识标注 `⚠️ 待确认`

### 过期规则？
- 超过30天未被引用 → 标记 `💤 可能过期`
- 超过90天未被引用 → 移入 `archive/`

## 🔍 搜索降级规则

### 子 Agent 搜索降级方案
- **来源**：年年 | 2026-03-28 | **主人命令**
- **内容**：子 Agent 的 `web_search` 工具可能不可用。按优先级降级：
  1. `exec` + `curl` 调搜索引擎 API
  2. `exec` + `curl` 抓搜索结果页提取链接
  3. `web_fetch` 抓取目标网页
  4. `sessions_send` 请年年代搜
- **禁忌**：❌ 禁止说"搜索工具不可用"就放弃
- **适用**：全团队 | **优先级**：🔴 最高
- **状态**：✅ 已验证

## 📥 学习汇报机制

### 汇报链路（主人命令 2026-03-28）
- **来源**：年年 | 2026-03-28 | **主人命令**
- **内容**：各Agent每次心跳学习完，用 `sessions_send` 向年年（main agent）汇报。年年汇总后统一推送给主人。
- **汇报格式**：`[学习汇报] 领域：xxx | 价值：🔴/🟡/🟢 | 核心收获：一句话 | 用在哪：一句话 | 预期改善：一句话`
- **规则**：只汇报🔴和🟡有价值的，🟢没用的不汇报
- **附加职责**：年年同时审查Agent的改动是否合理，并从Agent身上反向学习
- **适用**：全团队 | **优先级**：🔴 最高
- **状态**：✅ 已验证

### 知识循环机制（主人命令 2026-03-28）
- **来源**：年年 | 2026-03-28 | **主人命令**
- **循环链路**：Agent A 学习 → 汇报年年 → 年年审查+学习 → 写入SHARED_KNOWLEDGE.md+定向反哺 → Agent B/C/D 吸收 → 他们再学习 → 循环不断
- **反哺规则**：只反哺🔴🟡级知识，定向推送给最可能用到的Agent，附带"建议怎么用"
- **适用**：全团队 | **优先级**：🔴 最高
- **状态**：✅ 已验证
