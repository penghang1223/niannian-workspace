# KNOWLEDGE_FLOW.md - Agent 间知识流动协议

> 创建时间：2026-03-27
> 维护者：年年（main agent）
> 目标：Agent 间直接知识交换，不再全靠 main 中转

---

## 🔄 知识流动架构

### 旧模式（现状）
```
Agent A → main → SHARED_KNOWLEDGE.md → main → Agent B
         (中转)                       (中转)
```
问题：main 成为瓶颈，延迟高，效率低

### 新模式（目标）
```
Agent A → lessons.md → 心跳同步 → SHARED_KNOWLEDGE.md → Agent B
         (直接写)     (自动)     (共享)              (直接读)
```
优势：去中心化，延迟低，并行化

---

## 📡 知识交换协议

### 1. 写入规则

**谁可以写？**
- 所有 Agent 完成任务后都可以写入自己的 `lessons.md`
- main agent 负责同步到 `SHARED_KNOWLEDGE.md`

**写入格式**
```markdown
## [日期] [任务类型] - [Agent名]

### 经验内容
- 具体经验

### 适用范围
- 其他哪些 Agent 可以受益

### 优先级
- 🔴 高优先级（立即同步）
- 🟡 中优先级（下次心跳同步）
- 🟢 低优先级（有空再同步）
```

### 2. 读取规则

**谁应该读？**
- 所有 Agent 每次任务前读取 `SHARED_KNOWLEDGE.md`
- 特别关注自己领域的更新

**读取策略**
- 按角色过滤：只读与自己职责相关的部分
- 按时间过滤：优先读取最近7天的更新
- 按优先级过滤：优先读取 🔴 高优先级内容

### 3. 知识验证

**验证机制**
- 有争议的知识标注 `⚠️ 待确认`
- main agent 有最终决定权
- 验证过的知识标注 `✅ 已验证`

**验证流程**
```
Agent A 写入经验
    ↓
其他 Agent 阅读后提出质疑
    ↓
main agent 仲裁
    ↓
标注为 ✅ 已验证 或 ❌ 已否决
```

## 🔗 跨Agent协作增强

### 协作模式定义

#### 模式 1：知识共享
```
本尔发现一个测试模式
    ↓
写入 lessons.md（标注适用范围：开发工程师）
    ↓
心跳同步到 SHARED_KNOWLEDGE.md
    ↓
开发工程师下次任务时读取
    ↓
应用到代码实现中
```

#### 模式 2：经验传递
```
开发工程师踩了一个坑
    ↓
写入 lessons.md（标注：所有Agent注意）
    ↓
心跳同步到 SHARED_KNOWLEDGE.md
    ↓
夕尔读取后避免同样的坑
```

#### 模式 3：协作优化
```
娜尔写了一个PRD模板
    ↓
标注：本尔、开发工程师可参考
    ↓
本尔根据PRD模板设计测试用例
    ↓
开发工程师根据PRD模板实现功能
```

## 📊 知识流动指标

### 流动性指标
- **写入频率**：各 Agent 每周写入 lessons.md 次数
- **读取频率**：SHARED_KNOWLEDGE.md 被引用次数
- **同步延迟**：从写入到可被读取的时间
- **知识复用**：同一知识被多个 Agent 引用次数

### 质量指标
- **知识准确率**：已验证知识 / 总知识
- **知识完整度**：各领域知识覆盖率
- **知识新鲜度**：最近30天知识占比

## ⚠️ 知识流动边界

### 可以流动
- ✅ 工作经验、最佳实践
- ✅ 坑点教训、避坑方法
- ✅ 模板、工具、方法论
- ✅ 协作改进建议

### 不能流动
- ❌ 主人隐私信息
- ❌ 未验证的猜测
- ❌ 互相矛盾的结论
- ❌ 超出职责范围的建议
