# 多 Agent 协作小说工作流 v1.0

> 设计：年年 🎀 | 日期：2026-04-10
> 目标：用多 Agent 分工协作，实现高质量长篇小说创作

---

## 角色分工

| 角色 | Agent | 职责 | 不可做 |
|------|-------|------|--------|
| **planner** | 灵犀 | 结构规划、节奏设计、伏笔调度 | 不写正文，不审查 |
| **writer** | 惊鸿 | 正文生产、章节撰写 | 单章最多改 2 次，仅自审字数 |
| **checker** | 鉴微 | 质量审查、一致性检查 | 有最终否决权，不直接改正文 |
| **manager** | 年年 | 跨角色统筹、状态管理、主人汇报 | 不介入创作细节 |

---

## 工作流（章节级）

```
主人指令 → 年年分发 → 灵犀规划 → 惊鸿撰写 → 鉴微审查
                                         ↓ 失败？
                                    ← 最多返修 2 次
                                         ↓ 通过
                                   年年更新状态 → 交付
```

### Phase 1: 规划（灵犀）
**输入**：主人指令 / 章节号 / 当前 state.json
**输出**：章节 spec（结构化规划）

**spec 内容**：
- 本章核心事件（1 句话）
- 必须推进的伏笔（从 foreshadowing_engine 读取）
- 角色状态变化（从 emotion_states 读取）
- 节奏目标张力值（从 pace_controller 读取）
- 必须避免的套路（从 plot_memory 黑名单读取）
- 钩子/悬念设计（章末留什么钩子）

### Phase 2: 撰写（惊鸿）
**输入**：灵犀的 spec + state.json + 前 3 章摘要
**输出**：章节正文（Markdown）

**撰写规则**：
- 按 spec 执行，不偏离核心事件
- 遵守 style_controller 的文风要求
- 遵守 tell_show_blacklist 的禁词表
- 遵守 emotion_states 的情感跳格规则（最多跳一格）
- 遵守 plot_memory 的套路冷却期（10 章内不重复）
- 字数：3000-5000 字/章
- 钩子密度：每 500 字 ≥1 个钩子

### Phase 3: 审查（鉴微）
**输入**：惊鸿的正文 + 灵犀的 spec + state.json
**输出**：审查报告（通过/返修/重写）

**Checklist-α（10 项）**：
1. ✅ 字数达标（3000-5000 字）
2. ✅ 核心事件完成（对照 spec）
3. ✅ 伏笔注入/回收正确
4. ✅ 人物状态一致（无 OOC）
5. ✅ 情感跳格合规（最多跳一格）
6. ✅ 套路冷却期合规（10 章内不重复）
7. ✅ 禁词表无命中（tell_show_blacklist）
8. ✅ 章节结构完整（开头 + 发展 + 高潮 + 悬念）
9. ✅ 节奏张力在目标范围内
10. ✅ 钩子密度达标（每 500 字 ≥1 个）

**审查结果**：
- **通过** → 进入 Phase 4
- **返修**（≤3 项轻微问题）→ 退回惊鸿修改（最多 2 次）
- **重写**（≥4 项或严重问题）→ 退回灵犀重新规划

### Phase 4: 交付（年年）
**输入**：鉴微通过的正文
**输出**：
1. 更新 state.json（章节数 +1，状态更新）
2. 更新 plot_memory（记录本章使用的套路）
3. 更新 foreshadowing_engine（伏笔状态迁移）
4. 更新 pace_controller（记录实际张力值）
5. 更新 emotion_states（关系变化）
6. 提取内容资产（金句/标题/社媒帖子）
7. 向主人汇报

## 状态管理（state.json）

```json
{
  "meta": {
    "title": "书名",
    "author": "惊鸿",
    "total_chapters": 250,
    "completed_chapters": 105,
    "current_chapter": 106,
    "status": "active",
    "last_updated": "2026-04-10T18:55:00+08:00"
  },
  "characters": { ... },
  "foreshadowing_engine": { ... },
  "emotion_states": { ... },
  "plot_memory": { ... },
  "pace_controller": { ... },
  "style_controller": { ... }
}
```

**事务性更新**：
- 每次章节通过后，原子性更新所有状态
- 如果更新失败，回滚到上一版本
- 每次更新备份到 `state-backups/` 目录

---

## 失败处理

| 失败类型 | 处理策略 |
|---------|---------|
| 灵犀规划偏离 spec | 年年指出，灵犀重新规划 |
| 惊鸿撰写 OOC | 鉴微打回，惊鸿修改（最多 2 次） |
| 惊鸿 2 次修改仍不通过 | 退回灵犀重新规划 |
| 鉴微审查漏放 Bug | 年年抽检发现，鉴微记录教训 |
| state.json 更新失败 | 回滚到备份，人工修复 |

## 主人交互

### 主人可以：
- **给指令**："写第 106 章" / "推进 3 章" / "加强感情线"
- **看进度**：随时问"写到哪了"
- **给反馈**：第 X 章哪里不好 → 反馈入库 → 后续章节自动调整
- **审稿件**：随时查看任意章节
- **改设定**：直接修改 state.json → 后续章节自动加载新设定

### 主人不需要：
- ❌ 管理 Agent 分工
- ❌ 检查审查结果
- ❌ 更新状态文件
- ❌ 跟踪伏笔回收
