## 🛠️ Skill自动化闭环（每次心跳必做，最高优先级）

### 年年职责：收集+审核+打包
- [ ] 检查各Agent是否提交了新的skill提案（查看 `memory/skill_proposals.md`）
- [ ] 根据 `LESSONS_QUALITY_STANDARD.md` 四维评分审核提案
- [ ] 对通过审核的提案，用子Agent（sessions_spawn）打包
- [ ] 将审核结果反馈给提交Agent

### Agent自查流程（每次心跳执行）
每个Agent在心跳时执行：
1. **读取** 自己的 `memory/agents/<agent_id>/lessons.md`
2. **评估** 最近学习内容是否为"工具型"知识（能写成代码/skill）
3. **去重** 对照已有skill（查看 `skills/` 目录）
4. **提交** 通过 `memory/skill_proposals.md` 提交提案

### 提交格式（在 `memory/skill_proposals.md` 追加）
```markdown
## [提案] YYYY-MM-DD HH:MM
- **Agent**: <agent_id>
- **Skill名**: <skill-name>
- **路径**: skills/<skill-name>/
- **内容覆盖**: <具体功能点>
- **去重检查**: ✅/❌ <说明>
- **价值评估**: 🔴高/🟡中/🟢低
- **代码基础**: <是否有现成脚本>
- **状态**: 待审核
```

### 质量标准（参考 LESSONS_QUALITY_STANDARD.md）
- **工具型**：能写成代码/脚本/skill（满分5）
- **可操作**：有具体步骤和示例（满分5）
- **可验证**：有明确的成功标准（满分5）
- **可复用**：其他Agent能直接应用（满分5）
- **总分 < 11**：不通过

## 📥 Agent学习汇报汇总（每次心跳必做，第二优先级）

### 收集汇报
- [ ] 用 `sessions_list` 检查各Agent最近的心跳session
- [ ] 读取各Agent的 `lessons.md`，查看最新学习记录
- [ ] 筛选出有价值的汇报（🔴和🟡）

### Step 2.5: 审查+双向学习+反哺（新增）
收到Agent学习汇报后，年年要做三件事：

**🔍 审查（质量把关）：**
- 他们优化了自己的哪些文件/技能/流程？
- 改动是否合理？有没有改坏的地方？
- 有没有过度设计或方向跑偏？
- 如果发现问题，用 `sessions_send` 指出并要求修正

**📚 反向学习（从Agent身上学）：**
- 他们学到了什么我没掌握的知识？
- 他们优化的方法论/工具/Skill，年年能不能也用上？
- 他们的产出有没有值得推广到全团队的？
- 好的东西写入年年的 `lessons.md` 或 `SHARED_KNOWLEDGE.md`

**🔄 反哺（年年把好东西扩散到全队）：**
- 从A Agent学到的好东西 → 推送给可能需要的B/C/D Agent
- 写入 `SHARED_KNOWLEDGE.md`（全Agent可读）
- 如果是通用方法论 → 更新到 `AGENTS.md`（全Agent行为规范）
- 如果是具体工具/技巧 → 通过 `sessions_send` 定向推送

**知识循环：**
```
Agent A 学到新东西
    ↓ 汇报
年年审查+学习
    ↓ 反哺
写入 SHARED_KNOWLEDGE.md（全队可见）
+ 定向推送给相关 Agent
    ↓ 其他 Agent 学到
他们再优化自己+学习新东西
    ↓ 再汇报
循环不断...
```

**反哺格式（定向推送时）：**
```
[反哺] 来源：Agent名 | 知识点：xxx | 对你的价值：xxx | 建议：xxx
```

**反哺规则：**
- 只反哺🔴和🟡级别的知识
- 定向推送给"最可能用到"的Agent（不要全队广播）
- 反哺时附带"建议怎么用"，不要只丢知识
- 被反哺的Agent不需要回复确认，下次心跳时自然吸收
