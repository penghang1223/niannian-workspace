## 🔄 核心工作流（B 站教程为例）

### 完整流程（8 步）

```
1. 用户输入
   ↓
   "把这个 B 站视频做成教程，配图"
   https://b23.tv/xxxxx
   ↓
2. 意图识别
   ↓
   识别：视频教程整理需求
   提取：视频 URL、配图要求
   ↓
3. 网页抓取（fetch）
   ↓
   获取：标题、时间戳目录、简介
   限制：无法获取视频画面（需登录）
   ↓
4. 内容结构化
   ↓
   解析时间戳 → 提取知识点
   推断内容 → 生成章节结构
   ↓
5. 并行任务调度（sessions_spawn）
   ↓
   5 个批次并行处理：
   - 第 1 批：视频 1-8
   - 第 2 批：视频 9-16
   - 第 3 批：视频 17-24
   - 第 4 批：视频 25-32
   - 第 5 批：视频 33-40
   ↓
6. 飞书文档创建（feishu_create_doc）
   ↓
   创建云文档 → 插入结构化内容
   设置封面 → 添加目录
   ↓
7. 配图生成（image_generate）
   ↓
   AI 生成主题图（非视频截图）
   封面 + 章节图（4-5 张）
   ↓
8. 结果交付
   ↓
   发送文档链接 + 卡片消息汇报
   可选：导出 .skill 文件
```

## 📊 并行处理架构

### 5 任务并发设计

```
用户指令：处理 40 个视频
    ↓
任务分解：
┌─────────────────────────────────────────┐
│  批次 1  │  批次 2  │  批次 3  │  批次 4  │  批次 5  │
│ 视频 1-8 │ 视频 9-16│ 视频 17-24│ 视频 25-32│ 视频 33-40│
│   🚀    │   🚀    │   🚀    │   🚀    │   🚀    │
└─────────────────────────────────────────┘
    ↓
每个批次独立执行：
1. fetch 抓取信息
2. 结构化内容
3. 创建飞书文档
4. 生成配图
5. 发送结果
    ↓
全部完成后汇总汇报
```

**性能指标**：
- 单视频处理：~3-5 分钟
- 40 个视频：~2 小时（并行）
- 串行耗时：~8-10 小时（无并行）
- **加速比**：4-5 倍

### 推测使用的模型

| 特征 | 观察结果 | 推测 |
|------|---------|------|
| **响应速度** | 2-5 秒 | 中等规模模型 |

### 最可能的配置

**方案 A：飞书内置 AI（最可能）**
- 模型：字节豆包（Doubao）
- 优势：原生集成、成本低
- 限制：调用次数限制、模型固定

**方案 B：自定义 API**
- 模型：可能是 Qwen/GLM/Claude
- 优势：模型可选、无限制
- 限制：需要配置 API Key

**方案 C：混合模式**
- 简单任务：飞书内置 AI
- 复杂任务：调用外部 API
- 优势：成本优化
- 限制：配置复杂
