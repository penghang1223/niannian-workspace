## 🎨 设计风格分析

### 视觉特征

| 特征 | 说明 | 示例 |
|------|------|------|
| **标题** | 加粗 + emoji | `**10 个技能包已发送！** 🎉` |
| **表格** | 3 列为主，对齐清晰 | `\| # \| Skill \| 功能 \|` |
| **列表** | 短横线 + 空格 | `- 每个视频 → 获取信息` |
| **分割线** | `---` | 用于分隔内容块 |
| **emoji** | 每卡片 2-5 个 | 🚀 ✅ ❌ 🎉 ✨ 😊 |
| **语气** | 可爱 + 亲切 | "～"、"啦"、"呀" |

### 语言风格

```
✅ 正确示例：
- "完成后会分批自动发给你～"
- "文件已经发给你了，下载后可以在 aily 的 skill 管理中导入～ 😊"
- "预计完成后会分批收到通知，请耐心等待～"

❌ 避免示例：
- "完成后将分批发送给您。"（太正式）
- "文件已发送，请下载。"（太生硬）
- "请耐心等待通知。"（太冷漠）
```

## 📊 卡片使用统计

### 各类型使用频率

| 类型 | 使用次数 | 占比 |
|------|---------|------|
| 进度汇报 | 15+ | 30% |
| 方案选择 | 8+ | 16% |
| 完成通知 | 10+ | 20% |

| 错误说明 | 5+ | 10% |

### 出现场景

| 场景 | 卡片类型 | 触发条件 |
|------|---------|---------|
| 开始处理任务 | 进度汇报 | 用户确认方案后 |
| 需要用户选择 | 方案选择 | 多方案可选时 |
| 任务完成 | 完成通知 | 所有子任务完成后 |
| 发送 Skill | 技能列表 | .skill 文件发送后 |
| 遇到限制 | 错误说明 | 无法完成请求时 |
| 文档生成中 | 生成进度 | 后台处理时 |
| 文档完成 | 文档完成 | 飞书文档创建后 |

## 🚀 年年整合方案

### 短期（1 周）

**创建标准化卡片模板库**：

```python
# card_templates.py
CARD_TEMPLATES = {
    "progress_report": {...},
    "choice_selection": {...},
    "completion_notification": {...},
    "skill_info": {...},
    "error_explanation": {...},
    "generation_progress": {...},
    "document_completion": {...}
}

def send_card(template_name, **kwargs):
    template = CARD_TEMPLATES[template_name]
    content = render_template(template, **kwargs)
    await feishu_send_card(content)
```
