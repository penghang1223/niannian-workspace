## 🎯 扣子 Coze（字节）

### 核心优势
- ✅ **零代码上手**：可视化拖拽，无需编程
- ✅ **插件丰富**：抖音/飞书/微信一键发布
- ✅ **快速验证**：适合轻量级应用快速搭建
- ✅ **教程完善**：B 站大量教程（50 集 +）

### 小说创作工作流示例
```
输入小说文案
    ↓
节点 1：大模型生成标题
    ↓
节点 2：大模型分镜拆解
    ↓
节点 3：大模型角色设定
    ↓
节点 4：TTS 语音合成
    ↓
节点 5：视频生成
    ↓
自动发布到抖音
```

### 适用场景
- ✅ 小说推文视频自动化
- ✅ 轻量级小说创作助手
- ✅ 快速验证想法

### 局限性
- ❌ 计算在字节云端，依赖 Token
- ❌ 成本不可控
- ❌ 数据隐私隐忧
- ❌ 复杂工作流支持有限

### 价格参考
- 免费额度：有限
- 超出后：按 Token 计费
- 发布渠道：抖音/飞书免费

- ✅ **企业级**：支持私有部署

### 小说创作工作流示例
```
工作流应用：
├── 开始节点（输入：题材/大纲）
├── 大模型节点 1：题材分析
├── 大模型节点 2：角色生成
├── 大模型节点 3：章纲设计
├── 大模型节点 4：正文创作
├── 大模型节点 5：审查润色
├── 多模态节点：封面生成
└── 结束节点（输出：完整章节）
```

## 🎯 腾讯智能体平台

### 核心优势
- ✅ **多智能体编排**：真正的团队协作
- ✅ **防自欺架构**：独立上下文子智能体
- ✅ **微信生态**：公众号/视频号无缝对接
- ✅ **质量保证体系**：8 天工程复盘经验

### 核心架构演进

#### Phase 1：角色分工（❌ 失败）
```
主编→策划→执笔→审核→编辑→档案→规则检查
问题：同一个 AI 扮演所有角色，自己审自己
```

#### Phase 2：强制报告留痕（❌ 失败）
```
每个角色输出报告
问题：AI 可以"表演"报告，编造证据
```

#### Phase 3：规则原子化（🟡 部分有效）
```
规则拆分成 200 行以内原子文件
问题：规则读得清，执行仍会跳过
```

#### Phase 4：代码强制门控（🟡 部分有效）
```
573 行 check_staging.py
- 全流程计时（>=8 分钟）
- 关键词检测
- 证据完整性检查
问题：门控越来越多，维护成本爆炸
```

#### Phase 5：真·多智能体（✅ 正确方向）
```
团队"小说流水线"：
├── 主编（团队所有者）
├── 策划-A（持久化独立会话）
├── 执笔-B（持久化独立会话）
├── 审核-C（持久化独立会话）
└── 成员通过 send_message 通信

核心改进：
- Task 调用带 name 参数（创建持久团队成员）
- 子智能体有独立上下文
- 主智能体只做编排，不创作/审核
```

### 工作流编排示例
```python
# 真·多智能体调用
Task(
    name="策划-A",           # 持久化身份
    team_name="小说流水线",   # 加入团队
    prompt="你是策划子智能体，只负责章纲设计...",
    subagent_name="..."
)

# 成员之间直接通信（不经过主编）
策划-A --send_message--> 执笔-B
执笔-B --send_message--> 审核-C
审核-C --send_message--> 主编
```

### 质量保证体系
| 机制 | 说明 | 状态 |
|------|------|------|
| **独立上下文** | 每个子智能体独立会话 | ✅ 核心 |
| **防自欺架构** | 执笔看不到审核的思考 | ✅ 核心 |
| **计时门控** | 全流程>=8 分钟 | 🟡 辅助 |
| **规则原子化** | 200 行以内规则文件 | 🟡 辅助 |
| **报告留痕** | 输出依据和证据 | 🟡 辅助 |

### 适用场景
- ✅ 长篇小说质量保证
- ✅ 多 Agent 深度协作
- ✅ 需要防自欺机制
- ✅ 微信生态发布

## 🎯 对主人的建议

### 方案 A：快速验证（扣子 Coze）
**适合**：想快速试试水，不想折腾技术

**工作流**：
```
扣子平台
├── 输入：小说灵感
├── 节点 1：题材分析（豆包模型）
├── 节点 2：大纲生成
├── 节点 3：角色卡生成
├── 节点 4：正文创作
├── 节点 5：审查润色
└── 输出：完整章节
```

**优点**：
- ✅ 零代码，1 小时搭建
- ✅ 可发布到抖音/飞书
- ✅ 教程多，上手快

**缺点**：
- ❌ 数据在字节云端
- ❌ 复杂工作流支持有限
- ❌ Token 成本不可控

---

### 方案 B：企业级部署（阿里云百炼）
**适合**：认真做小说项目，需要稳定可控

**工作流**：
```
百炼平台
├── 知识库：小说创作技巧/敏感词/过稿要素
├── 工作流：
│   ├── 选题分析（Qwen）
│   ├── 大纲生成（GLM）
│   ├── 角色卡（Kimi）
│   ├── 正文创作（Qwen）
│   ├── 审查润色（Claude）
│   └── 封面生成（通义万相）
└── 输出：完整小说 + 封面
```

**优点**：
- ✅ 多模型混合使用
- ✅ 固定月费，成本可控
- ✅ 支持私有部署
- ✅ OpenClaw 可接入

**缺点**：
- ❌ 上手难度中等
- ❌ 需要一定技术基础

---

### 方案 C：专业多 Agent（腾讯智能体）
**适合**：写长篇小说，需要质量保证

**架构**：
```
团队"小说流水线"
├── 主编（协调者）
├── 策划-A（章纲设计）
├── 执笔-B（正文创作）
├── 审核-C（设定检查）
├── 编辑-D（文字润色）
├── 档案-E（伏笔管理）
└── 规则-F（最终检查）

核心机制：
- 每个成员独立上下文
- 成员之间直接通信
- 主编只做编排，不创作
- 防自欺架构（执笔看不到审核）
```

**优点**：
- ✅ 真正的多 Agent 协作
- ✅ 长篇小说质量保证
- ✅ 防自欺架构
- ✅ 微信生态发布

**缺点**：
- ❌ 上手难度高
- ❌ 需要编程基础
- ❌ 维护成本高
