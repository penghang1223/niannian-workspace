## 🎯 核心概念

### Claude Code 是什么？
**定义**：Anthropic 官方 CLI 工具，终端中的 AI 编程助手

**核心理念**：
- ✅ 终端优先（无需 GUI）
- ✅ 文件操作安全（沙箱机制）
- ✅ 多文件理解（完整项目上下文）
- ✅ 工具调用（exec/read/write/search）

## 🏗️ ECC 架构（12 层递进）

### 1. 核心循环
```
用户输入 → 规划 → 执行 → 验证 → 输出
```

### 2. 工具调度
**工具类型**：
- 📖 Read（读取文件）
- ✍️ Write（写入文件）
- 🔧 Bash（执行命令）
- 🔍 Grep（搜索内容）
- 🌐 WebFetch（抓取网页）

### 3. 规划系统
**规划层级**：
- 宏观规划（项目级）
- 中观规划（文件级）
- 微观规划（函数级）

### 4. 子代理系统
**子代理类型**：
- 规划代理

- 代码审查代理
- 安全审查代理

### 5. 知识加载
**知识类型**：
- 项目知识（本地代码库）
- 领域知识（技术文档）

### 6. 上下文压缩
**4 层压缩机制**：
| 层级 | 方法 | 效果 |
|------|------|------|
| snip | 裁剪无关内容 | 节省 30% |
| microcompact | 合并重复内容 | 节省 60% |
| collapse | 折叠已知内容 | 节省 40% |
| autocompact | 全量压缩 | 节省 50%+ |

## 💡 Karpathy 的使用方式

### 从"写代码"到"建大脑"

**传统用法**：
```
用户提问 → AI 写代码 → 对话结束 → 知识丢失 ❌
```

**Karpathy 用法**：
```
用户丢素材 → AI 整理成 Wiki → 持续积累 → 随时查询 ✅
```

### 三文件夹架构
```
llm-wiki/
├── raw/          # 原始素材（论文/代码/文章）
├── library/      # AI 整理的知识库
└── output/       # 产出内容（报告/代码/文章）
```

## 📋 最佳实践

### 1. 提示词技巧
**好提示词**：
```
"创建一个 Python 函数，实现快速排序算法。
要求：
1. 带类型注解
2. 包含 docstring
3. 有单元测试
4. 处理边界情况"
```

**坏提示词**：
```
"写个排序"
```

---

### 2. 上下文管理
**大项目处理**：
```bash
# 先让 AI 了解项目结构
claude "列出项目所有文件和目录结构"

# 再针对具体文件操作
claude "修改 src/user.py 的登录逻辑"
```

---

### 3. 安全使用
**权限控制**：
```bash
# 只读操作（安全）
claude --read-only "分析这个项目"

# 写入操作（需要确认）
claude "修改配置文件"
```

---

### 4. 效率优化
**批量操作**：
```bash
# 一次性描述所有需求
claude "
1. 创建用户模型
2. 添加 CRUD 接口
3. 写单元测试
4. 生成 API 文档
"
```
