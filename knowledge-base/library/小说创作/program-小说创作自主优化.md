# 📚 小说创作自主优化指南

> 版本：v1.0  
> 创建时间：2026-04-08  
> 灵感来源：Karpathy Autoresearch  
> 适用：多 Agent 小说创作系统

---

## 🎯 目标

**改进小说正文以提升完读率/读者评分**

---

## 📋 核心规则

### 1. 每次只改一个东西
- ✅ 开篇（前 300 字）
- ✅ 爽点位置（第 1000/2000/3000 字）
- ✅ 节奏（对话/描写比例）
- ✅ 人设（主角性格/台词风格）
- ✅ 情节（冲突强度/反转密度）

**禁止**：一次改多个东西（无法归因）

---

### 2. 记录改动原因
每次改动前必须写：
```markdown
## 实验记录 #001

**改动内容**：开篇从"穿越"改为"对话"

**改动原因**：
- 知乎盐选数据显示对话开篇完读率高 15%
- 参考《全家反派》对话开篇成功案例

**预期效果**：完读率提升 10%+
```

---

### 3. 发布测试（进度驱动）
- 发布到知乎盐选/番茄小说
- **收集 1000 次阅读** 或 **50 个付费用户**
- 核心指标：完读率/付费率/读者评分

---

### 4. 数据驱动决策
| 指标 | 决策 |
|------|------|
| 完读率提升≥10% | ✅ 保留，继续优化 |
| 完读率提升 5-10% | 🟡 保留，观察长期 |
| 完读率变化<5% | ❌ 回滚，换方向 |
| 完读率下降 | ❌ 立即回滚，记录教训 |

---

## 🔄 实验循环（进度驱动）

```
1. 读取当前小说正文
   ↓
2. 分析数据（完读率/付费率/评论）
   ↓
3. 提出假设（"如果改 X，完读率会提升"）
   ↓
4. 修改正文（只改一个东西）
   ↓
5. 发布测试（收集 1000 次阅读或 50 个付费）
   ↓
6. 收集数据
   ↓
7. 决策（保留/回滚）
   ↓
8. 记录实验日志
   ↓
9. 重复循环
```

## 📊 实验日志模板

```markdown
## 实验 #001

**日期**：2026-04-08

**改动内容**：开篇从"穿越"改为"对话"

**改动原因**：
- 知乎盐选数据显示对话开篇完读率高 15%
- 参考《全家反派》对话开篇成功案例

**预期效果**：完读率提升 10%+

**实际效果**：
- 完读率：45% → 52%（+15.6%）✅
- 付费率：8% → 9.5%（+18.8%）✅
- 读者评分：4.2 → 4.5（+7.1%）✅

**决策**：✅ 保留

**经验教训**：
- 对话开篇确实有效
- 下次可以尝试"对话 + 冲突"组合
```

## 📈 评估指标

### 核心指标（必须追踪）
| 指标 | 目标值 | 计算方式 |
|------|--------|---------|
| **完读率** | ≥50% | 读完人数/点击人数 |
| **付费率** | ≥10% | 付费人数/读完人数 |
| **读者评分** | ≥4.5 | 平均星级评分 |

### 辅助指标（参考）
| 指标 | 说明 |
|------|------|
| 评论数 | 读者互动热度 |
| 收藏数 | 长期价值认可 |
| 分享数 | 内容传播力 |

## 🚀 实验方向建议

### 开篇优化
- [ ] 对话开篇 vs 穿越开篇
- [ ] 冲突前置（300 字内）vs 铺垫前置
- [ ] 第一人称 vs 第三人称
- [ ] 现在时 vs 过去时

### 爽点优化
- [ ] 爽点密度：每 1000 字 vs 每 2000 字
- [ ] 爽点类型：打脸 vs 逆袭 vs 装逼
- [ ] 爽点强度：小爽 vs 大爽
- [ ] 爽点位置：章末 vs 章中

### 节奏优化
- [ ] 对话比例：60% vs 40%
- [ ] 描写密度：详细 vs 简略
- [ ] 章节长度：2000 字 vs 3000 字
- [ ]  cliffhanger：每章都有 vs 隔章有

### 人设优化
- [ ] 主角性格：咸鱼 vs 努力
- [ ] 台词风格：现代梗 vs 古风
- [ ] 金手指类型：系统 vs 老爷爷
- [ ] 成长速度：快 vs 慢

---

## ⚠️ 注意事项

### 1. 控制变量
- 每次只改一个东西
- 其他因素保持不变
- 确保数据可归因

### 2. 样本量
- 至少 1000 次阅读
- 至少 24 小时测试期
- 避免偶然性

### 3. 外部因素
- 避开节假日（数据异常）
- 避开热点事件（流量波动）
- 记录发布时间

### 4. 伦理底线
- 不刷数据
- 不误导读者
- 保持内容质量
