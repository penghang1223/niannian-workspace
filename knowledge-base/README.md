## 🔄 通用工作流程

```
1. 收集素材（任何领域）
   ↓
   丢入 raw/对应分类
   ↓
2. AI 整理
   ↓
   阅读 raw/素材
   提取知识点
   创建/更新 library/文章
   建立反向链接
   ↓
3. 创作产出
   ↓
   基于 library/知识库
   创作 output/内容
   ↓
4. 反馈循环
   ↓
   新反馈/新学习 → raw/
   → 更新 library/
   → 优化 output/
```

## 📋 命名规范

### raw/ 素材命名
```
<日期>_<领域>_<描述>.md
示例：
- 2026-04-08_小说_修仙系统题材分析.md
- 2026-04-08_技术_OpenClaw 架构学习.md
- 2026-04-08_学习_Karpathy 第二大脑.md
- 2026-04-08_工作_项目会议纪要.md
```

### library/ 文章命名
```
<主题>.md
示例：
- 修仙 + 系统.md
- OpenClaw 架构.md
- 费曼学习法.md
- 高效会议指南.md
```

### output/ 产出命名
```
<类型>/<项目>-<版本>.md
示例：
- 小说/《我在修仙界开网约车》-v1.0.md
- 代码/爬虫工具-v1.0.py
- 文章/Karpathy 学习报告-v1.0.md
- 报告/2026-Q1 工作总结.md
```
